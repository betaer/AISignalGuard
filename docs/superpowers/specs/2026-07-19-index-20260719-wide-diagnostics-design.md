# `index-20260719.html` 诊断区宽度统一设计

## 目标

以线上根页面 `index.html` 为唯一产品基线创建 `index-20260719.html`。日期版沿用根页全部功能与资源，并将“出口 IP（风控第一顺位）”以及其后的全部诊断区块，从当前 720px 内容宽度扩展到与上方网络风险与身份分析内容一致的 960px。`demo/index-new.html` 不作为本轮设计或实现基线。

## 版式边界

- 上方结果区域由 `#identity-result-root` 承载，当前宽度为 `min(100%, 960px)`。
- “出口 IP”及后续诊断由 `.analysis-workspace > #section-root` 承载，当前宽度为 `min(100%, 720px)`。
- 新页面通过独立的 `<body>` 类名 `dated-wide-diagnostics` 启用宽版诊断布局。
- 新增作用域样式：

  ```css
  body.dated-wide-diagnostics .analysis-workspace > #section-root {
    width: min(100%, 960px);
  }
  ```

- 原 `index.html` 不增加该类名，因此继续保持现有 720px 诊断宽度。

## 响应式规则

- 桌面端：`#identity-result-root` 与 `#section-root` 左右边界一致。
- 平板与移动端：两者都受外层 `.shell` 和 `min(100%, 960px)` 约束，不产生页面级横向溢出。
- 日期版不单独修改卡片内部网格、字号、触控目标、导航或浮动工具栏；根页与日期版共同使用本轮更新后的网络连通逻辑。

## 全站文字下划线规范

- `重新选择目标身份` 默认显示下划线，不再只在 hover 时出现。
- `评分规则`、`隐私说明`、通用分析入口、页脚文字链接以及状态链接使用统一几何参数：`1px` 粗细、`4px` 文字偏移。
- 中性链接使用 `var(--line-2)` 浅灰下划线，hover 时随文字适度加深。
- AI 状态等语义链接继续使用 `currentColor`：绿色状态配绿色下划线、黄色状态配黄色下划线、红色状态配红色下划线；只统一间距和粗细，不抹掉状态语义。
- 触控目标仍可保持 32px 或 44px 高，但下划线通过 `text-decoration` 贴近文字，不再使用控件外框底边模拟，避免控件越高时线条离文字越远。

## 结果区数量文案

- 嵌入各检测分区顶部的身份信号摘要只显示数量，例如“1 项信号”。
- 删除“点击查看依据”，避免在证据已经位于当前展开区域时重复提示或产生错误的折叠交互暗示。
- 真实的可展开 `<details>` 仍保留箭头、状态和键盘交互，不改变其展开能力。

## 网络连通服务矩阵

- 通用数字身份分析：`Google.com`、`YouTube.com`、`WhatsApp.com`、`Reddit.com`；移除 ChatGPT。
- 自媒体创作者：`TikTok.com`、`YouTube.com`、`Instagram.com`、`X.com`，同时保留画像评分需要的广告平台探针。
- 跨境商家：`Shopify.com`、`Amazon.com`、`PayPal.com`、`Stripe.com`。
- AI 用户：继续展示 AI 服务与补充工具探针；只有 ChatGPT、OpenAI、Claude、Gemini、Perplexity 五项核心服务阻塞身份结论，Cursor、GitHub、npm 作为后台补充项，不延迟正式结果。
- 全球站点与中国站点补齐面向用户的规范域名写法，例如 `Google.com`、`QQ.com`、`BiliBili.com`、`Baidu.com`、`TaoBao.com`。
- 所有服务均使用浏览器只读 GET 探针，不接入付费接口、不携带账号凭据、不读取业务正文；使用三个站点 worker，并为主探针切换官方备用端点预留一个网络槽，使浏览器层峰值仍不超过四路。当前画像和大陆连通判定所需服务优先执行；运行中禁用“重测”，连续激活也只保留一个 worker 池，避免叠加第三方请求。
- 可读取 CORS 2xx 的结果显示“可达 · Nms”；opaque 跨站响应显示“有响应 · Nms”，表示浏览器收到响应但无法读取 HTTP 详情，不代表服务、账号或地区访问受限。
- 可读取的 CORS 非 2xx 显示“HTTP NNN · 服务响应异常 · Nms”，保留已知服务故障，不降级成“未确认”，也不继续 fallback 掩盖故障。
- Nms 是浏览器从发起请求到取得响应头的耗时；如果启用备用端点，则计至备用端点终态。取得所需信号后立即中止未读取的正文传输。该耗时包含 DNS、TLS、服务端响应与浏览器调度，不是完整下载耗时，也不等同于 ICMP Ping，更不代表区域解锁、账号或支付能力。
- 探针使用 `cache: no-store`，不追加随机缓存穿透参数，避免人为制造 CDN 缓存未命中并扭曲耗时。
- 主地址优先使用标签所展示服务的同域官方轻量页面或静态资源；只有网络、CORS 或超时导致主探针无法取得响应时才尝试官方备用静态资源，并在结果中明确标注“官方备用资源”。

## 验收标准

1. `index-20260719.html` 可独立打开并完成通用数字身份分析。
2. 1440px 桌面端中，`#section-root` 与 `#identity-result-root` 的左右边界差值不超过 1px。
3. 原 `index.html` 的 `#section-root` 仍保持 720px，不被日期版样式影响。
4. 720px、390px 与 300px 视口均无页面级横向溢出，诊断卡片完整落在视口内。
5. `重新选择目标身份`、评分/隐私入口和页脚链接默认下划线均为 1px、偏移 4px；状态链接的下划线颜色与状态文字相同。
6. 各检测分区显示“数字 + 项信号”，不再包含“点击查看依据”。
7. 源 CSS/JS 与压缩产物一致，生产构建与 Sites Worker 测试通过。
8. 发布后 GitHub Pages 返回 200，线上 HTML/CSS/JS 与本地提交一致。
9. 根页网络连通同时展示通用、AI、自媒体创作者和跨境商家服务；通用组不再包含 ChatGPT。
10. 服务结果区分“可确认 2xx”“请求已建立但响应状态不可读”“可读取的 HTTP 异常”三类，展示连接到响应头耗时，并明确不把结果解释为解锁、账号或支付状态。
11. 连续快速触发网络连通重测时，主探针重复请求为 0，实际活跃请求峰值不超过四路。
12. AI 用户的核心五项服务完成后即可生成正式结论，慢速补充开发工具继续后台更新而不阻塞结果页。
