# 移动端关键信息首屏设计

## 目标

让正式首页在常见手机可用视口中，检测前无需滚动即可看完三个目标画像和开始入口；检测后无需滚动即可看完风险结论、唯一网络参考分和六类风险信号。完整证据、判定口径、评分规则与旧检测能力继续保留在下方，不删数据、不改变算法。

## 首屏信息层级

### 检测前

1. 品牌栏。
2. “你想验证哪种数字身份？”与两行以内的用途说明。
3. AI 用户、自媒体创作者、跨境商家三个紧凑单选行。
4. 自动倒计时/开始分析主按钮。
5. “跳过选择，使用通用数字身份分析”入口。

手机端隐藏卡片中的“重点分析”和标签胶囊；名称、目标描述与选中状态保持可见。标签仍保留在 DOM 中并在平板、桌面正常显示。

### 检测后

1. 品牌栏与完整横向检测导航；十个检测项目直接列出，可左右滑动，当前项目自动保持在可见区域。
2. 当前画像、高级网络风险诊断、风险结论与风险项计数。
3. 唯一的网络信号参考分。
4. 出口 IP、身份、多源互证、AI 出口、泄漏、网络连通六类信号。
5. 一句总结和最多两个最高优先级风险提示；风险气泡作为一组居中排列，换行后每行继续居中。

判定口径、评分规则、隐私说明以及完整身份证据继续位于首屏之后。重新选择目标身份保留 44px 触控面积。

## 响应式策略

- `621px` 以上保持现有桌面/平板布局。
- `620px` 以下启用专用手机首屏布局：身份卡变为紧凑列表，十项结果导航使用 44px 高的横向可滑动栏，风险头部与圆环压缩。
- `340px` 以下沿用“分数圆环 + 2×3 信号网格”，保留两枚最高优先级风险胶囊；底部操作栏进入文档流，避免覆盖评分、风险提示或操作入口。
- 高度不超过 `400px` 的手机横屏同样让底部操作栏进入文档流；横向导航固定在顶部并保持可滑动，避免与操作栏或检测内容重叠。
- 使用 `100dvh` 仅作为尺寸参考，不用固定高度裁切页面；Safari 地址栏伸缩后页面仍可正常滚动。
- 工具栏继续保留五个操作：常规手机使用底部浮动形态并遵守安全区；超窄屏和手机横屏切换为静态形态，与风险圆环、节点和文本操作保持可视间距。

## 视觉密度

- 手机品牌栏约 44px 高。
- 检测前标题 28px 左右，说明 13px；身份行 68–76px，行间距 8px；主按钮至少 44px。
- 检测后标题 22–24px；风险轨道约 270–286px；节点至少 44×44px；总结最多两行；首屏风险胶囊最多两枚。
- 不通过缩到 10px 以下的正文来换空间；非关键说明改为下一屏，而不是牺牲可读性。

## 可访问性

- 隐藏的标签只属于补充说明，身份名称、描述、单选控件及按钮均保持可访问。
- 身份选项、品牌入口、导航、风险节点、风险胶囊、判定按钮、文本按钮与五个浮动操作均保持至少 44×44px 可触区域。
- 不改变标题层级、结果完成播报、六节点按钮、键盘焦点和 tooltip 逻辑。
- 关键总结使用完整、简短的一句话，不用固定高度或 `line-clamp` 裁切；风险胶囊在手机端按行居中并完整换行，总数继续由风险计数明示。

## 验收标准

- 390×664 与 430×740：检测前第三张身份卡、主按钮和通用入口底部均位于视口内。
- 390×664 与 430×740：检测后完整横向导航、风险标题、结论、分数圆环和六类信号均位于视口内；浮动工具栏不与圆环或节点相交。
- 手机结果页：十个导航项目全部直接列在横向滚动容器中，不产生页面级横向溢出；活动项目随页面滚动保持可见。
- 风险气泡：单枚、多枚同排及多行换行时，气泡组均以内容区中轴居中。
- 300px 宽：无横向滚动，六类信号使用 2×3 网格，五项操作以单行静态工具栏呈现且可点击。
- 195px 宽（等效 200% 缩放）：检测前按钮和卡片不溢出；检测后品牌与导航不重叠，五项操作以 3+2 两行静态工具栏呈现。
- 1280px 与 1440px：身份卡、风险首屏、导航与浮动栏布局不退化。
- 源 CSS 与压缩 CSS 一致，GitHub Pages 缓存键更新。
