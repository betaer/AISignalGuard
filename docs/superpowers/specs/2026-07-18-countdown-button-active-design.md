# 倒计时主按钮主动状态设计

## 背景

首页未选择目标画像时会显示 6 秒倒计时，并在到期后自动进入通用数字身份分析。当前主按钮使用原生禁用态：浅灰背景、灰色文字、无阴影和 `not-allowed` 光标。虽然行为正确，但视觉上像不可执行的失效控件，无法表达“自动分析已经排队并正在倒计时”。

## 设计目标

- 倒计时状态明显区别于失效状态，并与绿色分析进度条属于同一视觉体系。
- 视觉上具有轻微材质感，但不引入高饱和、外发光或持续动画。
- 不制造假的可点击暗示。既然主按钮呈现为主动状态，用户点击时应立即开始通用数字身份分析。
- 选择目标画像后继续使用现有深色主按钮，保证所选画像操作的优先级更高。
- 重新选择且倒计时已经消费后，未选择画像的按钮恢复真正的禁用态。

## 方案比较

### A. 仅加深禁用态

改动最少，但控件仍不可点击。视觉表现与真实交互语义冲突，用户会尝试点击一个没有响应的按钮，因此不采用。

### B. 主动倒计时按钮，可点击立即开始通用分析

采用。倒计时期间按钮启用，使用比进度条更浅的鼠尾草绿背景、深绿色文字、细绿色边线、内高光和轻微染色阴影。点击与倒计时到期都复用 `startIdentityAnalysis("generic")`，并由现有阶段保护避免重复启动。

### C. 把按钮改造成纯进度指示器

语义最清晰，但会削弱首页主要操作，也与用户明确提出“按钮颜色更深”的方向不一致，因此不采用。

## 状态规则

| 状态 | 可点击 | 文案 | 视觉 |
|---|---:|---|---|
| 初次进入且倒计时进行中 | 是 | `开始分析所选身份 (6s)` 至 `(1s)` | 柔和鼠尾草绿主动状态 |
| 已选择目标画像 | 是 | 现有画像专属文案 | 现有深色主按钮 |
| 重新选择后未选画像、倒计时不再重启 | 否 | `开始分析所选身份` | 现有灰色禁用态 |
| 分析启动后 | 不显示 | 不适用 | 进入分析进度页 |

## 配色与材质

- 背景：`oklch(0.74 0.095 148)` 到 `oklch(0.70 0.10 148)` 的轻微纵向渐变，比进度条 `--green` 更浅。
- 边线：接近进度条色相的中深绿色。
- 文字：深森林绿，确保与浅绿色背景达到 WCAG AA 对比度。
- 阴影：一层顶部内高光、一层短距离绿色染色阴影，不使用发光效果。
- Hover：略微加深并上移 1px；Active：回落到原位，形成轻触反馈。
- Focus：保留项目现有清晰焦点轮廓。

## 无障碍与语义

- 倒计时期间移除 `disabled`，因为点击确实会触发立即分析。
- 通过 `data-auto-countdown="true"` 暴露稳定的样式与测试状态，不依赖按钮文案选择器。
- 屏幕阅读器继续通过现有 `aria-live` 状态获知自动进入规则。
- 倒计时取消后移除主动状态属性，避免残留错误样式。

## 验证范围

- E2E 断言倒计时按钮已启用、具有主动状态属性，并且点击后立即进入通用分析。
- 断言连续点击或与计时器竞争时只启动一次核心检测。
- 断言选择画像后主动倒计时状态被清除，按钮保持画像专属状态。
- 断言重新选择后按钮恢复真实禁用态。
- 桌面与移动端截图检查颜色、文字对比、阴影、Hover、Active 和换行。
- 运行完整检查，重新生成压缩 CSS，并更新缓存版本号。
