# A / C 静态演示页设计规格

日期：2026-07-16  
状态：待书面确认

## 1. 目标

在仓库 `/demo/` 目录新增两个相互独立的精简 HTML，用于保留并比较已经确认的 A、C 两套首屏排版：

- `demo/identity-passport.html`：A「数字身份护照」。
- `demo/network-pass.html`：C「网络登机牌」。

两个页面是视觉参考稿，不接入检测逻辑，不替换 `index.html`，也不进入正式评分流程。

## 2. 共同内容

两页使用同一组明确标注为示例的数据，方便只比较布局：

- 身份：`通用数字身份分析`。
- 目标画像：`通用数字环境`。
- 身份匹配分：`84 / 100`。
- 状态：`部分接近`。
- 脱敏出口 IP：`172.59.186.x`。
- 地区：`JP · Tokyo`。
- ASN：`AS21928`。
- 网络类型：`Mobile / T-Mobile`。
- 隐私状态：`示例数据 · 已脱敏`。

页面必须显示“视觉演示 · 示例数据”，避免访问者把静态内容误认为实时检测结果。

## 3. A：数字身份护照

整体是一张深绿色圆角身份卡：

- 左上为 `DIGITAL IDENTITY PASSPORT`、身份名称与目标画像。
- 右上为薄荷绿色环形 `84` 分数。
- 中部使用细分隔线。
- 下部左侧显示出口 IP，右侧显示琥珀色“部分接近”状态。
- 最底部使用胶囊标签显示地区、ASN 与网络类型。
- 背景保留低对比度同心圆装饰，但不得影响文字对比度。

视觉重点是简洁、完整、适合手机一屏截图。

## 4. C：网络登机牌

整体是一张暖白色票据：

- 顶部黑色窄栏显示 `AISG / NETWORK PASS` 与 `LOCAL REPORT`。
- 第一层左侧为大号 `84`，右侧为身份名称、目标画像与“部分接近”状态。
- 第二层独占显示脱敏出口 IP。
- 第三层三列显示地区、ASN 与网络类型。
- 各层使用虚线分隔，形成票据凭证感。
- 底部包含明确标注为装饰的校验纹和“示例数据 · 已脱敏”。装饰校验纹设置 `aria-hidden="true"`，不得暗示可以扫码。

视觉重点是规整、专业、像一张可分享的诊断凭证。

## 5. 文件与技术边界

- 每个 HTML 都是完整、自包含的静态文档，CSS 内联于自身文件。
- 不加载 `app.js`、`app.min.js`、第三方字体、远程图片或任何 API。
- 不写入 Cookie、Local Storage、剪贴板或分析服务。
- 两页均设置 `meta robots="noindex, nofollow"`，并以正式主页作为 canonical。
- 顶部 `AI Signal Guard` 品牌链接指向 `https://betaer.github.io/AiSignalGuard/`，不带下划线。
- 不新增 `demo/index.html`，本轮严格输出用户要求的两个 HTML。

## 6. 响应式规则

- 页面从 `320px` 开始可用，卡片宽度使用视口减安全边距，并设置合理最大宽度。
- 手机端确保主要卡片在常见 `390 × 844` 视口内完整可见并适合截图。
- 桌面端将卡片居中展示，不无意义拉伸。
- `320 / 360 / 390 / 768 / 1440px` 均不得产生页面级横向滚动。
- 长 IPv6、ASN 或网络类型必须允许自然换行；极窄屏下事实栏由三列转为两列，网络类型占整行。
- 尊重安全区和 `prefers-reduced-motion`；演示页不依赖动画传达任何状态。

## 7. 无障碍

- 使用 `main`、`article`、标题和 `dl` 等语义结构。
- 分数必须有完整文本标签“身份匹配分 84 / 100”，不能只依赖大数字。
- 状态不能只依赖颜色，同时显示“部分接近”。
- 所有装饰图形均从辅助技术中隐藏。
- 文字与背景保持可读对比度，键盘可访问品牌链接。

## 8. 验收

实现后进行以下验证：

1. 两个文件可直接从本地静态服务器打开，无控制台脚本错误、无外部网络请求。
2. 两页示例内容一致，只有视觉组织不同。
3. `320 / 360 / 390 / 768 / 1440px` 下 `scrollWidth === clientWidth`。
4. `390 × 844` 下核心卡片无需滚动即可完整截图。
5. A 保留深绿护照、右上环形分数和底部胶囊事实；C 保留黑色顶栏、虚线票据分区和底部校验纹。
6. 仓库现有 `index.html`、检测逻辑、构建产物与测试不得因演示页发生变化。
