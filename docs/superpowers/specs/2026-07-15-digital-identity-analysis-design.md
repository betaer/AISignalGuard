# AiSignalGuard 数字身份匹配分析设计规格

日期：2026-07-15  
状态：已确认，进入实现

## 1. 产品定位

AiSignalGuard 从单纯展示网络参数，升级为分析“互联网如何识别当前数字环境”的匹配工具。产品不判断用户真实身份，只比较浏览器、网络与服务路径信号和目标数字画像之间的相似程度。

主流程：

1. 选择目标数字身份。
2. 在用户明确点击后运行环境检测。
3. 将已有检测证据映射到目标画像。
4. 输出 Identity Match Score、证据覆盖率、自然语言总结与可追溯原因。
5. 保留完整底层检测证据供高级核对。

## 2. 交互状态机

页面包含 `select`、`running`、`result` 三个公开状态：

- `select`：只展示四张画像卡和通用分析入口，不运行 IP、DNS、WebRTC、服务路径等检测。
- `running`：展示当前目标与检测进度，调用原有完整检测链路。
- `result`：显示身份总结、信号、原因、匹配详情、建议和完整检测证据。

重新检测沿用当前目标画像；重新选择画像返回选择页。无论重新检测还是切换画像后再次开始，均以同一页面内的干净检测事务重新运行完整链路：旧请求会被中止、旧响应会被令牌作废，不刷新文档，也不复用可能已经变化的网络证据。

## 3. 身份画像

系统内置五个画像（四个可选目标画像和一个通用画像）：

- `generic`：通用数字身份分析。
- `us_consumer`：美国普通用户。
- `tiktok_creator`：TikTok 创作者。
- `cross_border_seller`：跨境卖家。
- `ai_worker`：AI Worker。

每个画像包含 `id`、`name`、`icon`、`description`、`target`、`weights`、`checks`、`serviceIds` 和 `criticalRules`。每个画像的权重总和必须为 100，并由自动化测试校验。

## 4. 证据模型与评分

每个检测项输出：

- `status`：`match`、`partial`、`mismatch` 或 `unknown`。
- `confidence`：0 到 1 的证据可信度。
- `evidence`：面向用户的证据说明。
- `source`：底层信号来源。

评分使用画像动态权重，而非等权平均。单项状态基础值为匹配 1、部分匹配 0.6、不匹配 0、未知 0.5；可信度使结果向中性值 0.5 回归：

`effective = 0.5 + (baseValue - 0.5) × confidence`

`score = Σ(weight × effective) ÷ Σ(weight) × 100`

证据覆盖率独立计算：

`coverage = Σ(weight × confidence) ÷ Σ(weight) × 100`

在核心证据未就绪前不展示最终分数。明确的关键矛盾可使用画像中声明的透明分数上限，并必须在“为什么不像”与匹配详情中显示原因。未知信号不归入负向原因，而进入“尚未确认”。

## 5. 结果信息架构

结果区顺序固定为：

1. 身份总结卡：目标画像、分数、覆盖率、一句话总结。
2. 身份信号分析：Location、Network、Timezone、Language、Browser、DNS、WebRTC 和服务路径。
3. 为什么像 / 为什么不像 / 尚未确认。
4. 目标画像匹配详情：每项权重、状态、证据与贡献。
5. 如何更接近目标画像：仅基于已确认差异生成环境一致性建议。
6. 完整检测证据：保留现有所有检测区块。

## 6. 服务检测边界

浏览器可直接观察到的语言、时区、WebRTC、DNS 和已返回的 IP 情报可作为环境证据。TikTok、Netflix、PayPal、Stripe 等跨站服务在无官方接口时只能形成“可达性或访问路径信号”，不得解释为区域内容解锁、账户可用或交易能力。

跨域限制、浏览器拦截和超时统一显示为未知，不得显示为成功或失败。项目不新增付费接口。

## 7. 文案与隐私

- 所有结论描述“环境”“特征”“信号”“匹配度”“相似度”或“数字画像”。
- 不输出国籍、职业或真实个人身份判断。
- 地区称谓必须有可追溯证据；只有确认州级位置后才生成州级总结。
- 需求列明的四个禁用词通过自动化检查防止进入用户可见文案。
- 检测开始前不发送检测类请求；原有第三方访问说明继续保留。

## 8. 兼容与回归

原有出口 IP、IPv4/IPv6、ASN、ISP、DNS、WebRTC、浏览器、语言、时区、AI 服务、网络路径、多数据源和报告复制能力必须保留。旧风险评分可作为内部证据适配来源，但不再作为首屏公开主分数。

## 9. 响应式与无障碍

- 画像选择使用语义化单选控件和明确的开始按钮。
- 桌面端与平板使用双列 2×2 布局，移动端使用单列布局。
- 选中卡片只显示一条深色边线；键盘焦点由单选框自身的 `focus-visible` 焦点环表达。
- 身份信号分析在桌面端使用双列信息卡，移动端使用单列，长证据在卡片内自然换行。
- “尚未确认”仅在确实存在待确认信号时显示；没有待确认项时不渲染空状态区域。
- 支持键盘、清晰焦点、状态播报、减少动画偏好。
- 360 像素宽度不得出现页面级横向滚动。

## 10. 分享与隐藏状态

- 右下角按顺序提供全部重测、ChatGPT、Claude、复制给 AI、复制分享文案、隐私模式、GitHub / Star 和返回顶部八个动作；全部使用内联 SVG，并兼容桌面与移动端。窄屏把 GitHub / Star 独立为工具栏上方胶囊，其他七项保持单行。
- ChatGPT 与 Claude 是独立、安全的新窗口快捷入口；“复制给 AI”只复制结构化 Markdown 诊断报告，不自动打开或向任何服务发送内容。
- GitHub / Star 从顶部迁入右下角快捷区，使用低频、低幅度的图标呼吸变色，并在系统要求减少动态效果时停用动画。
- AI 诊断报告无条件脱敏，不受页面隐私模式开关影响：IPv4 保留前三段并将最后一段替换为 `x`；IPv6 保留前三组并将剩余组替换为 `xxxx`；mDNS、Canvas 与声纹标识不写入原值。
- “复制分享文案”继续使用现有 280 加权字符内的宣传摘要；不提供 TXT、图片分享或 JSON 下载。
- 隐私模式仅控制页面敏感字段的显示，并通过 `aria-pressed`、文字、标题和视觉共同表达持久状态。
- 复制按钮具有复制中、成功和失败状态；成功状态保持两秒后回到默认状态。
