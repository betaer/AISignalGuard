# 身份入口受众调整设计

## 目标

首页只展示三个更宽泛、易理解的目标画像，并按用户优先级排列：

1. AI 用户
2. 自媒体创作者
3. 跨境商家

美国普通用户画像继续保留在配置和评分模块中，但暂不作为首页可选入口展示。

## 命名与内容

### AI 用户

- 卡片描述：面向国际 AI 服务的常见使用环境。
- 重点信号：AI 服务路径、ChatGPT / Claude、Gemini / Perplexity、网络信誉、DNS / WebRTC。
- 结果页目标名称：国际 AI 服务用户。
- 评分以 ChatGPT、OpenAI、Claude、Gemini、Perplexity 对应产品域名的浏览器可达性为核心。官方状态页、通用静态资源以及 Cursor、GitHub、npm 等补充证据不参与评分，相关检测能力仍保留在产品的网络检测模块中。

### 自媒体创作者

- 卡片描述：面向美国市场的内容创作与社交平台环境。
- 重点信号：网络与信誉、TikTok / Instagram、YouTube、广告平台信号、地理一致性。
- 结果页目标名称：面向美国市场的自媒体创作者。
- 继续使用 TikTok、Instagram、YouTube 和广告平台信号，但不再用单个平台定义画像名称。

### 跨境商家

- 卡片描述：面向美国市场的跨境经营与商业服务环境。
- 重点信号：IP 信誉、Shopify / Amazon、PayPal / Stripe、浏览器环境、位置一致性。
- 结果页目标名称：面向美国市场的跨境商家。

## 兼容策略

- 保留 `ai_worker`、`tiktok_creator`、`cross_border_seller`、`us_consumer` 内部 ID，避免历史链接、报告和测试引用发生无意义迁移。
- `us_consumer` 配置、权重和检测服务保持不变，仅入口卡片使用 HTML `hidden` 属性暂时隐藏，并放在卡片列表最后。
- 通用数字身份分析入口继续保留。

## 布局与响应式

- 桌面宽度大于 900px：三列一行。
- 621px 到 900px：两列布局。
- 620px 及以下：单列布局。
- 继续保留现有单层深色选中边框、键盘焦点环、悬停和点击反馈。

## 测试与发布

- 单元测试验证五个内部画像仍完整，并验证 AI 用户不再以开发者生态为评分条件。
- E2E 验证可见卡片数量、顺序、桌面三列、平板两列、移动端单列、隐藏的美国画像不可交互，以及三个画像能正常完成分析。
- 重建 `app.min.js` 与 `styles.min.css`，提升静态资源缓存版本。
- 完整执行 `npm run check:full` 后合并到 `main` 并推送，由 GitHub Pages 从 `main` 根目录自动发布。
