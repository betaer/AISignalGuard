# 根首页高级网络风险诊断首屏化设计

## 目标

将正式首页结果页末尾的“高级网络风险诊断”提升为首屏主结论，并直接替换当前占据顶部的大号“数字环境综合结论”卡片。页面只保留一套风险诊断 DOM，不复制评分、节点或交互逻辑。

## 最终信息顺序

1. 高级网络风险诊断：默认完整展示圆环参考分、六类风险节点、风险结论与计数。
2. 身份匹配依据：保留“为什么像 / 为什么不像 / 尚未确认”。
3. 调整建议：继续使用当前身份画像动态生成的建议。
4. 原有出口 IP、身份信号、泄漏、网络连通、多源交叉、AI 路径等检测模块。

原来的大号身份总结、摘要句、证据覆盖率和页面末尾的第二个诊断入口不再展示。

## DOM 与渲染架构

- `#identity-result-root` 改为稳定的静态结果容器，继续兼容既有 `#identity-result-root` 链接。
- 唯一的 `#sec-score` 静态风险模块移入该容器顶部，避免运行时移动节点或复制内部 ID。
- 新增 `#identity-result-content`，`renderIdentityResult()` 只更新此动态容器，因此多次渲染不会销毁风险圆环、控制按钮或节点事件。
- `#sec-score` 从折叠的 `<details>` 改为始终可见的语义 `<section>`；评分规则和隐私说明仍由现有按钮独立展开。
- 风险分算法、原子风险分类、复制报告和六节点数据源保持不变。

## 身份画像信息

风险诊断标题区显示当前分析上下文：

- 通用模式：`通用数字身份分析 · 不预设地区或职业`
- 目标画像：`当前目标：画像名称 · 目标画像描述`

风险诊断标题区保留“重新选择目标身份”，身份依据标题栏只显示紧凑的当前画像标签。正式首页不再显示第二套目标匹配百分制；网络圆环是全页唯一的数字评分，并始终明确标注为“网络信号参考分”。按照附件图，圆环显示分数本身而不额外堆叠 `/100`；复制报告仍使用 `N/100` 表达。身份匹配算法继续用于生成“为什么像 / 为什么不像”和调整建议，但不与网络参考分并列展示。

分散到出口 IP、网络泄漏、网络连通等真实检测模块中的身份信号卡继续保留定性状态与证据，用于解释画像依据；不再展示“权重”或“得分贡献”等数字，避免形成第二套评分语义。

## 导航与焦点

- 顶部导航第一项保留 `#identity-result-root`，名称改为“网络风险”。
- 删除独立的 `#sec-score` 导航项，避免两个锚点指向同一首屏区域。
- 分析完成后的焦点落到风险诊断主标题，旧的 `#identity-result-root` URL 仍落在新首屏顶部。
- 风险诊断使用唯一 H1，身份依据和建议继续使用 H2。

## 视觉与响应式

- 风险诊断使用原身份总结的 960px 主容器宽度和一致的纸白、浅绿、浅黄配色。
- 恢复附件中的圆环数字分视觉，隐藏临时的“风险信号 / 点击节点查看”占位核心。
- 桌面使用 460px 轨道以匹配 960px 首屏容器；移动端沿用 330px，并在 340px 以下将六节点重排为 2×3 网格。
- 标题区在 720px 以下纵向堆叠，风险结论左对齐；不得产生横向滚动。

## 验收标准

- 结果页首个可见主模块是完整的高级网络风险诊断。
- `.identity-summary-card` 不再生成。
- 页面只有一个 `#sec-score`、一个 `#score-number` 和一组 `#score-nodes`。
- 风险分、节点、风险计数、评分规则和隐私说明正常更新。
- 全页只有一个可见数字评分节点，且属于网络信号参考分；目标画像不再显示第二个数字分。
- 旧 `#identity-result-root` 链接、移动端导航、复制给 AI 报告和重新选择身份均正常。
- 桌面、390px、300px 与短横屏无横向溢出或浮动按钮遮挡。
